---
title: Building a boring task list
date: 2021-07-19
---

The interesting work gets attention and the foundational work gets avoided. The recurring check-in, the update that keeps everyone aligned, the small improvement that compounds: none of these feel like progress while you're doing them. [[LNO Framework]] would classify most of them as leverage rather than overhead, because they prevent larger failures rather than respond to them. Most real progress is built on exactly this boring layer.

[[Slam it shut and move on]] keeps the list moving rather than stalling on things that are done enough. [[Focus on outcome, not output]] is the check: the boring task earns its place not because it feels productive but because it changes something. A list that contains only the exciting work is a wish list.
