---
title: Accessibility is the first rule of functionality
date: 2021-07-15
---

If something can't be reached, it doesn't work for that person, regardless of how refined it looks to everyone else. The visual polish, the interaction details, the copy: none of it matters if the user can't get there. Accessibility gets framed as compliance or inclusion work, but it's the first condition of functionality, the layer beneath which nothing else operates. A feature that exists but can't be used is absent, and the hard part is that this absence hides because it looks fine to everyone who already has access.

This is the baseline that [[Solve the problem not the feature]] keeps returning to: the problem isn't the interface, it's whether the right person can use it at all. [[Create to give]] only holds up if the thing you built can be received, and [[Keep user experience discussions at the product level]] is where the real constraints on who can access what actually get decided.
